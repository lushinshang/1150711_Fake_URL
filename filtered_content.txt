[7/10 資安案例分享，使⽤ mac 及
claude code 朋友請留意 ]
最近新購的 MacBook Air M5 剛到貨 (7/9) ，正在把需要的軟體裝的差
不多了，剩下 Claude Code 還沒裝。 結果事情⼤條。 ---
7/10 上午我先去 google 搜尋最新版 claude code 安裝指令，第⼀個結果看起來是 claude.ai
的 domain 沒錯 ( 圖⼀ ) ，進去也有⼀⾏指令 ( 圖⼆ ) ，忙碌中⼀時不察就在新機執⾏ .... 忽然感到不太對勁，瞄⼀眼
網址怎麼是 base64 的，然後開始跳 macos 的 keychain ⼀直問我密碼！ ⾺上警覺，立刻中⽌、關閉
WIFI、關機、進入 Recovery 完整抹除磁碟。⾄此，忽然覺得這兩天新機做環境都做⽩⼯了 = = --- 從貼上指令執⾏到斷網與關機，總共過去約 40 秒。 採⽤處理過的 claude chat 內容 share 出來這招，果然⾼。 冷靜想了⼀下，
以原本的平常⽤⼯作機回頭分析可能有那些問題： #⼀ 將該指令的
base64 解碼後網址送去 VirusTotal 檢測，尚未被發現出問題 ( 圖三 ) #⼆ 打開 Claude
Code 請它以該指令模擬整個執⾏後以及⼜下載那些 payload 進⾏，結果分析快完成時 Claude 就因為資安政策打斷了
#三 改⽤ Codex
接續分析 #四 同時，⽤
Claude Desktop 接取我 mcp_graylog ，從 log 中分析 opnsesne 、 zenarmor 、 adguard 、等等記錄，確認下載兩次
payload 並上傳⼀次資料出去。 #五 接著搭配
mcp_pve 、 mcp_wazuh 、 mcp_zimbra 等等我寫的多個週邊系統 mcp ⼯具檢測，確認尚未發起橫向移動⾏為。
#六 該主機還沒安
裝 Wazuh Agent ，所以沒來得及記錄到本機⾏為。 #七 Codex 分析
完成，確認分三個階段：執⾏ script 、下載其它 script 執⾏並上傳搜集資料、下載另⼀⽀惡意程式並取得重點應⽤程式資料，其中
包括 Ledger/Trezor 這兩個加密貨幣相關軟體甚⾄會替換成加料版 ) #八 它會嘗試偷本
機密碼、Keychain 內所存的帳號密碼、主流瀏覽器帳號密碼記憶庫 /Cookie/IndexedDB.. 等、⼀堆系統維運與開發相關的配置
--- 新機剛開始初始化配置無太多資料，且我儘量不⽤ macos 的 keychain 與瀏覽
器記憶密碼，所以第⼀次那包出去的資料中應無重要系統資料，但為了保險起⾒，我仍然立刻檢視所有帳號隨即變更密碼、檢查登入
裝置、確認 2FA 。 --- 註⼀：Codex 可以完整跑完所有分析， Claude 稍
微看到可疑的東⻄就罷⼯不幹了 .... 幹。 ( 圖四 ) 註⼆：我剛剛看 (7/11) 這個搜尋結果依然還掛在
第⼀筆。( 圖五 ) 註三：在我的測試機驗證， Avast Free for Mac 版本已經可以攔下第⼀步、第
⼆步的惡意 script 下載。
